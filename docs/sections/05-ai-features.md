# 板块五：AI 功能

## 1. AI 配置

使用任何 AI 功能前，需要先配置 AI API。点击顶栏的 **「AI 配置」** 按钮进入配置页面。

![AI 配置页面](images/05-ai-config.png)

配置页面分为以下几个区域：

### 1.1 1Shell AI 引擎

页面中央展示 1Shell 内置的 AI 引擎说明。点击 **「配置 API」** 按钮设置：

| 字段 | 说明 | 示例 |
|------|------|------|
| **API 基础地址** | OpenAI 兼容 API 地址 | `https://api.openai.com/v1` |
| **API Key** | API 密钥 | `sk-xxxx...` |
| **模型名称** | AI 模型 | `gpt-4o` |

支持任何 OpenAI 兼容服务商（OpenAI、DeepSeek、OpenRouter 等）。

### 1.2 AI CLI 工具

页面下方展示可接入的 AI CLI 工具：

| 工具 | 说明 |
|------|------|
| **Claude Code** | Anthropic 官方 CLI，接入后可在 Agent 面板中使用 |
| **OpenAI Codex CLI** | OpenAI 官方 CLI |
| **OpenCode** | 开源 AI CLI 工具 |

每个工具卡片显示安装状态（已安装/未安装）和版本信息。

### 1.3 Bridge 配置（左侧栏）

| 字段 | 说明 |
|------|------|
| **Bridge API** | Bridge 端点地址（自动生成） |
| **MCP Server** | MCP SSE 端点地址（自动生成） |
| **Bridge Token** | 鉴权 Token（来自 `.env`） |

## 2. AI Chat（AI 运维助手）

AI Chat 是终端页面右侧的对话面板，用于在当前终端上下文中与 AI 交流。

![AI 运维助手](images/05-ai-chat.png)

### 2.1 功能特点

- **终端上下文感知**：AI 能看到你当前终端的内容，基于实际输出提供建议
- **流式输出**：AI 回复实时流式显示
- **多主机适配**：切换主机后 AI 自动切换上下文

### 2.2 使用方式

1. 在终端右侧展开 AI 运维助手面板
2. 在输入框中输入问题
3. 按 **Ctrl+Enter** 或点击「发送」

### 2.3 使用场景

- 「这个错误是什么意思？」— AI 结合终端输出给出解释
- 「怎么查看 nginx 的错误日志？」— 给出适合当前系统的命令
- 「帮我写一个备份脚本」— 生成针对当前主机环境的脚本

## 3. 1Shell AI（全局 AI 助手）

1Shell AI 是平台的核心 AI 能力，相当于一个 7×24 小时的 AI 运维工程师。

### 3.1 全局面板模式

点击右下角的 AI 图标或顶栏的 **「1Shell AI」** 按钮，展开全局 AI 面板。

![1Shell AI 全局面板](images/05-1shell-ai-global.png)

面板顶部功能栏：

| 按钮 | 功能 |
|------|------|
| **页面标签**（如「网站管理」） | 显示当前所在页面，AI 自动适配该页面的能力 |
| **停止** | 停止 AI 当前正在执行的操作 |
| **安全** | 安全模式开关，开启后 AI 的每步写操作需人工审批 |
| **清空** | 清除对话历史 |
| **关闭** | 收起面板 |

### 3.2 IDE 模式

在主控台或终端页面，1Shell AI 以 IDE 模式运行，拥有完整的服务器操作能力。

![1Shell AI IDE 模式](images/05-1shell-ai-ide.png)

- 标题显示 **「1Shell AI」** + **「IDE」** 标签
- 输入框提示：「描述你的需求，如：查看所有容器状态、检查磁盘占用...」
- 状态栏显示「待命」/「执行中」

### 3.3 AI 拥有的能力

1Shell AI 内置 23 个工具，可以：

- **在任意主机上执行命令**：不需要手动切换终端
- **读写文件**：查看配置、修改文件
- **管理主机**：查看主机列表、主机状态
- **创建运维程序**：自动生成 Program / Skill / Playbook
- **操作网站**：创建/修改反向代理站点、管理 SSL 证书
- **查看监控数据**：读取探针采集的指标

### 3.4 页面感知

1Shell AI 会根据当前所在的页面自动调整能力描述：

| 页面 | AI 能力 |
|------|--------|
| 主控台 | 多主机管理、命令执行、文件操作 |
| 网站管理 | 反向代理站点创建/修改/删除、SSL 证书管理 |
| 容器管理 | Docker 容器操作 |
| 程序页 | Program 创建与调试 |

### 3.5 安全模式

开启安全模式后（点击「安全」按钮），AI 的每一步写操作都会弹出审批条：

- 显示 AI 即将执行的命令或操作
- 用户点击「批准」后才会执行
- 点击「拒绝」则跳过该操作
- 适用于生产环境，防止 AI 误操作

## 4. AI 选区分析

在终端中框选一段输出文本，AI 自动分析并给出解读。

![AI 选区分析](images/05-ai-selection.png)

### 4.1 使用方式

1. 在终端中用鼠标框选一段文本（如错误日志、命令输出）
2. 右侧自动弹出「AI 选区分析」面板
3. AI 分析选中内容，给出解读和建议

### 4.2 分析内容

- **错误解读**：解释错误消息的含义
- **修复建议**：给出具体的修复命令
- **安全提示**：如发现安全更新未安装，提醒安装

## 5. Ghost Text（终端 AI 补全）

在终端输入命令时，AI 实时提供内联补全建议。

### 5.1 工作方式

1. 在终端开始输入命令
2. AI 根据输入内容和终端上下文，在光标后方显示灰色补全建议
3. 按 **Tab** 采纳建议
4. 继续输入则忽略建议

### 5.2 特点

- 实时响应，输入时自动触发
- 上下文感知，基于当前目录、历史命令提供建议
- 不干扰正常输入，建议以灰色文字显示

## 6. AI 命令建议

用自然语言描述需求，AI 生成可执行的 Shell 命令。

### 6.1 使用方式

1. 点击终端工具栏的 **「AI 命令」** 按钮
2. 在弹出的面板中用自然语言描述需求（如「查看占用 80 端口的进程」）
3. AI 生成对应的命令
4. 点击 **「注入终端」** 将命令写入终端
5. 按回车执行

### 6.2 优势

- 不需要记住复杂的命令语法
- AI 会根据当前操作系统生成正确的命令
- 生成的命令可以在注入前检查和修改

## 7. AI Agent 面板

AI Agent 面板可以在 1Shell 的 Web 界面中直接运行外部 AI CLI 工具。

![AI Agent 面板](images/05-ai-agent.png)

### 7.1 打开方式

点击顶栏的 **「AI Agent」** 按钮。

### 7.2 支持的 CLI 工具

| 工具 | 说明 |
|------|------|
| **Claude Code** | Anthropic 的 AI 编程助手 |
| **OpenAI Codex CLI** | OpenAI 的命令行工具 |
| **OpenCode** | 开源 AI CLI（需配置 API） |

通过下拉框切换不同的 CLI 工具。

### 7.3 操作按钮

| 按钮 | 功能 |
|------|------|
| **启动** | 在远程主机上启动 AI CLI |
| **本地启动** | 在 1Shell 本机上启动 AI CLI |
| **停止** | 终止运行中的 AI CLI |
| **清屏** | 清除输出 |

### 7.4 目标主机

面板顶部显示当前 Agent 的目标主机（如 `amd3600 (root@IP)`），AI CLI 将在该主机的环境中运行。

### 7.5 沙箱状态

状态栏显示沙箱就绪状态和配置来源，确保 AI CLI 在受控环境中运行。
