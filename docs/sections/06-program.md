# 板块六：运维程序（Program）

---

## 1. 概述

Program 是 1Shell 的核心能力——用声明式 YAML 定义长驻运维任务，按时间表自动在指定主机上执行，并配备三层 AI 引擎保障。

进入方式：左侧导航栏 → **「程序」**

![运维程序页面](images/06-program-page.png)

---

## 2. 页面布局

### 2.1 顶部统计栏

| 指标 | 说明 |
|------|------|
| **Program 总数** | 已创建的程序数量 |
| **总实例数** | 所有程序绑定的主机实例总数 |
| **活跃/运行中** | 当前启用并正在执行的实例数 |
| **近 24H 错误** | 最近 24 小时内执行失败的次数 |

### 2.2 左侧：程序列表

列出所有已创建的 Program，每个卡片显示：

- 程序名称
- 触发方式（cron / manual）
- 绑定的主机数
- 启用/停用状态（彩色标签）

点击程序卡片查看详情。

### 2.3 右侧：程序详情

选中程序后，右侧显示详细信息，分为多个标签页：

| 标签页 | 内容 |
|--------|------|
| **信息** | 程序名称、描述、YAML 配置 |
| **结果** | 最近一次执行的输出数据（render step 的展示） |
| **触发历史** | 所有执行记录，包含时间、状态、耗时 |
| **实时条件** | 当前 verify 条件的判定状态 |
| **L2 Skill** | 程序关联的 Skill 列表 |
| **L3 Guardian** | Guardian AI 的介入记录 |

## 3. 创建 Program

### 3.1 AI 创作（推荐）

1. 进入「创作工作台」（左侧导航 → 创作）
2. 用自然语言描述需求，例如：
   - 「每 5 分钟检查所有服务器的 CPU、内存、磁盘」
   - 「监控 nginx 进程，挂了自动重启」
   - 「每天凌晨 3 点清理 /tmp 目录」
3. AI 自动生成完整的 `program.yaml` 并写入文件

### 3.2 手动创建

在 `data/programs/<program-id>/` 目录下创建 `program.yaml` 文件。

---

## 4. 实例管理

每个 Program 绑定一台或多台主机，每台主机生成一个「实例」。

### 4.1 启用实例

程序详情 → 实例列表 → 点击目标主机旁的「启用」按钮。

启用后程序按 `triggers` 中定义的 cron 表达式自动执行。

### 4.2 手动触发

点击实例旁的「触发」按钮，立即执行一次（不受 cron 时间限制）。

### 4.3 自定义操作按钮

如果程序定义了 `ui.instance_actions`，实例卡片上会显示自定义按钮（如「立即检查」「强制重启」）。破坏性操作按钮为红色，点击后需确认。

---

## 5. 三层 AI 引擎

这是 1Shell 最重要的架构设计：

### 5.1 L1：确定性执行

- 运行预设的 Shell 命令（如 `cat /proc/stat`）
- 用 `verify` 规则判断成功/失败（退出码 + 正则匹配）
- **零 AI 消耗**，纯规则驱动

### 5.2 L2：AI 约束执行

- 当 L1 发现异常时，调用 Skill（AI 能力包）
- Skill 在限定范围内诊断和修复
- 通过 `when` 条件触发，避免无谓的 token 消耗
- **少量 AI 消耗**

### 5.3 L3：Guardian AI 全权介入

- L2 也无法修复时，Guardian AI 获得完整诊断权限
- 自主分析问题根因，执行修复
- 有 `max_actions_per_hour` 滑动窗口限制，防止 AI 失控循环
- **按需 AI 消耗**

### 5.4 设计理念

能用简单规则解决的不动用 AI，需要判断力的才让 AI 介入。既省成本，又保安全。

---

## 6. 查看结果

程序执行后，在「结果」标签页查看输出。支持四种展示格式：

| 格式 | 用途 |
|------|------|
| **keyvalue** | 键值对展示，适合单台主机的指标详情 |
| **table** | 表格展示，适合多台主机的指标对比 |
| **message** | 文本消息 |
| **list** | 列表展示 |
