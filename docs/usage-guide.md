# 板块一：登录与系统设置

---

## 1. 登录

### 1.1 访问面板

启动 1Shell 后，在浏览器中打开：

```
http://服务器IP:3301
```

如果是本机运行，直接访问 `http://localhost:3301`。

### 1.2 登录界面

登录页面包含两个输入框：

- **用户名** — 默认 `admin`
- **密码** — 默认 `admin`

输入后点击「登录」进入主界面。

### 1.3 首次登录注意事项

- **请立即修改默认密码**，方法见下方「账号设置」
- 如果通过 HTTP（非 HTTPS）+ 非 localhost 地址访问，部分浏览器可能会拦截 Cookie，导致登录后无法进入主界面。解决方案：
  - 推荐：配置域名 + SSL 证书，通过 HTTPS 访问
  - 临时：在浏览器设置中允许该站点使用 Cookie

### 1.4 暴力破解防护

同一 IP 连续登录失败 **5 次**后，自动锁定 **60 秒**，期间该 IP 的任何登录请求都会被拒绝。锁定到期后自动解除。

### 1.5 退出登录

点击右上角的「退出」按钮即可退出登录，会话立即失效。

---

## 2. 系统设置

点击主界面左下角的 **「⚙ 设置」** 按钮，弹出系统设置窗口。

设置窗口包含两个标签页：

### 2.1 账号设置

可以修改登录的用户名和密码。

| 字段 | 说明 |
|------|------|
| **用户名** | 新的登录用户名 |
| **新访问口令** | 新密码，留空表示不修改 |
| **确认新访问口令** | 再次输入新密码，两次必须一致 |

点击「保存设置」即时生效，**无需重启服务**。修改后会同时更新 `.env` 文件中的 `APP_LOGIN_USERNAME` 和 `APP_LOGIN_PASSWORD`。

### 2.2 IP 访问控制

控制哪些 IP 可以访问 1Shell 面板。

**白名单模式：**
- 开启后，只有名单中的 IP 可以访问
- 其他所有 IP 的请求都会被拒绝

**黑名单模式：**
- 开启后，名单中的 IP 无法访问
- 其他 IP 正常访问

两种模式可以同时使用。

**添加规则：**
1. 选择类型（白名单 / 黑名单）
2. 输入 IP 地址或 CIDR 格式网段（如 `192.168.1.0/24`）
3. 可选填写备注
4. 点击「添加」

已添加的规则在下方列表中展示，可随时删除。

---

## 3. AI API 配置

在主界面中找到 **AI 配置**入口（通常在 AI 功能区域附近），弹出 AI API 配置窗口。

| 字段 | 说明 | 示例 |
|------|------|------|
| **API 基础地址** | OpenAI 兼容 API 的地址，不含 `/chat/completions` | `https://api.openai.com/v1` |
| **API Key** | 对应服务商的 API 密钥 | `sk-xxxx...` |
| **模型名称** | 要使用的模型，可手动输入或点击「获取模型」自动拉取 | `gpt-4o` |

**兼容的 API 服务商：**
- OpenAI（官方）
- DeepSeek
- OpenRouter
- 任何兼容 OpenAI Chat Completions 格式的服务

点击「获取模型」按钮，会自动调用 API 拉取该服务商支持的模型列表，填入下拉框供选择。

配置完成后点击「保存」，所有 AI 功能（AI Chat、Ghost Text、命令建议、选区分析、1Shell AI、运维程序 AI 引擎）立即生效。

---

## 4. 环境变量配置（.env 文件）

除了在 Web UI 中配置，也可以直接编辑项目根目录下的 `.env` 文件。首次运行时会自动从 `.env.example` 复制一份。

### 4.1 完整变量说明

| 变量 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `APP_LOGIN_USERNAME` | 否 | 登录用户名 | `admin` |
| `APP_LOGIN_PASSWORD` | **建议** | 登录密码，生产环境必须修改 | `admin` |
| `APP_SECRET` | **是** | 会话加密密钥，用于 SSH 凭据的 AES-256-GCM 加密存储 | 无 |
| `APP_SESSION_TTL_HOURS` | 否 | 登录会话有效期（小时） | `12` |
| `PORT` | 否 | 服务监听端口 | `3301` |
| `OPENAI_API_BASE` | 否 | AI API 基础地址 | `https://api.openai.com/v1` |
| `OPENAI_API_KEY` | 否 | AI API 密钥 | 无 |
| `OPENAI_MODEL` | 否 | AI 使用的模型名称 | `gpt-4o` |
| `BRIDGE_TOKEN` | 否 | MCP / Bridge API 鉴权 Token，启用 MCP 时必填 | 无 |
| `BRIDGE_EXEC_TIMEOUT_MS` | 否 | Bridge 命令执行超时（毫秒） | `30000` |
| `AGENT_DEFAULT_PROVIDER` | 否 | 默认 AI Agent 类型 | `claude-code` |
| `LOG_LEVEL` | 否 | 日志级别：debug / info / warn / error | `info` |
| `PROBE_TIMEOUT_MS` | 否 | 探针超时（毫秒） | `12000` |
| `PROBE_INTERVAL_MS` | 否 | 探针轮询间隔（毫秒） | `15000` |
| `PROBE_REMOTE_CONCURRENCY` | 否 | 远程探针并发数 | `3` |
| `TRUSTED_PROXY_IPS` | 否 | 受信任反向代理 IP，用于正确获取客户端真实 IP | 无 |

### 4.2 修改方式

```bash
# 用任意文本编辑器编辑
nano .env      # Linux
notepad .env   # Windows
```

修改 `.env` 后需要**重启 1Shell 才能生效**（Web UI 中修改的账号密码除外，那个是即时生效的）。

### 4.3 安全提示

- `APP_SECRET` 请设置为 64 位以上的随机字符串，用于加密存储 SSH 凭据
- `APP_LOGIN_PASSWORD` 生产环境请务必修改，不要使用默认的 `admin`
- `BRIDGE_TOKEN` 如果启用了 MCP 功能，请设置为高强度随机字符串
- `.env` 文件包含敏感信息，不要提交到版本控制（已在 `.gitignore` 中排除）


# 板块二：主机管理

---

## 1. 主控台概览

登录后进入主控台，这是 1Shell 的首页。

![主控台](sections/images/02-host-dashboard.png)

顶部显示「主控台」标题和 **「+ 添加主机」** 按钮，下方是搜索框和所有主机卡片列表。

### 1.1 本机

第一张卡片是「本机」，代表 1Shell 服务所在的机器本身，始终存在，不可删除。

- 显示「本地 Shell 状态：ready」
- 点击「切换」进入本机终端
- 点击「编辑」修改本机名称

### 1.2 远程主机卡片

每台远程主机以卡片展示，包含：

- **主机名称**（自定义）
- **连接信息**：`用户名@IP:端口`
- **连接状态**：已连接 / 未连接
- **网站任意门**：关联的业务链接（如 1panel 管理后台）
- **操作按钮**：切换（进入终端）、编辑、删除

### 1.3 搜索

搜索框支持按主机名称或 IP 地址模糊搜索，快速定位目标主机。

### 1.4 安全提示

如果使用默认的 `APP_SECRET`，顶部会显示黄色警告：「当前使用默认 APP_SECRET，上线前请配置强随机密钥。」请在 `.env` 中设置一个 64 位以上的随机字符串。

---

## 2. 添加与编辑主机

点击主控台右上角 **「+ 添加主机」** 或已有主机卡片的 **「编辑」** 按钮，打开主机配置表单。

![添加/编辑主机](sections/images/02-host-edit.png)

### 2.1 基础信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **主机名称** | 自定义名称，方便识别 | 阿里云生产 |
| **用户名** | SSH 登录用户名 | root |
| **主机地址** | 服务器 IP 或域名 | 8.148.xxx.xxx |
| **端口** | SSH 端口 | 22 |

### 2.2 中继跳板（ProxyJump）

用于连接 NAT 内网中无法直接访问的机器。

- **直连（无跳板）**：默认选项，直接 SSH 连接目标主机
- **选择跳板机**：下拉选择一台已添加的公网主机作为跳板，1Shell 会先 SSH 到跳板机，再从跳板机 SSH 到目标内网机器

适用场景：目标机器在内网（如 `192.168.x.x`），只有一台公网机器可以访问它。

### 2.3 认证方式

两种认证方式可选：

**密码认证：**
- 选择「密码」标签
- 输入 SSH 密码
- 编辑时留空表示保持原密码不变

**私钥认证：**
- 选择「私钥」标签
- 粘贴 SSH 私钥内容（PEM 格式）
- 如果私钥有密码，还需填写密钥密码

所有凭据使用 AES-256-GCM 加密存储，不保存明文。

### 2.4 快捷跳转（网站任意门）

在同一个编辑表单的底部，可以为该主机添加多个业务链接。

每条链接包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| **链接名称** | 链接的显示名称 | 1panel |
| **链接地址** | 完整 URL（仅支持 http:// 或 https://） | https://8.148.xxx.xxx:8888 |
| **描述** | 链接说明 | 业务后台 |

- 点击「新增链接」添加一条新链接
- 点击「删除链接」移除已有链接
- 保存后，链接会显示在主控台该主机卡片的「网站任意门」区域，点击即可在新窗口打开

### 2.5 保存

填写完成后点击右下角 **「保存主机」**，主机卡片立即出现在主控台。保存后可以直接从左侧切换到该主机的终端。

---

## 3. 探针中心

点击主控台顶部的探针入口，进入探针中心页面。

![探针中心](sections/images/02-probe-center.png)

### 3.1 总览信息栏

页面顶部显示所有主机的汇总统计：

| 指标 | 说明 |
|------|------|
| **总主机数** | 已添加的主机总数 |
| **在线** | 探针正常采集的主机数 |
| **异常** | 探针采集失败或指标异常的主机数 |
| **离线** | 无法连接的主机数 |
| **最近更新** | 最近一次探针轮询的时间 |
| **轮询间隔** | 探针采集频率（默认 60 秒，可通过 `PROBE_INTERVAL_MS` 调整） |

### 3.2 单机探针卡片

每台主机一张卡片，实时展示以下指标：

| 指标 | 采集来源 | 说明 |
|------|---------|------|
| **网络流量** | `/proc/net/dev` | 上行/下行速率（B/s） |
| **延迟** | SSH 连接耗时 | 连接延迟（ms） |
| **CPU 使用率** | `/proc/stat` | 百分比 |
| **内存使用率** | `/proc/meminfo` | 百分比 |
| **磁盘使用率** | `df -P` | 百分比 |
| **最后更新** | — | 最近一次采集成功的时间 |

### 3.3 零侵入原理

1Shell 的探针**不需要在目标机器上安装任何 Agent**。

工作原理：通过已有的 SSH 连接，定期在目标机器上执行采集命令（如 `cat /proc/stat`），读取输出后在 1Shell 服务端解析。整个过程复用 SSH 链路，对目标机器零侵入。

### 3.4 异常与离线

- **异常**（橙色）：SSH 连接成功但采集命令执行失败，或返回数据异常
- **离线**（红色）：SSH 连接失败（如超时 `Timed out while waiting for handshake`）
- 主机离线时会实时告警

### 3.5 操作按钮

每张探针卡片底部有两个按钮：

- **查看数据**：查看该主机的详细指标数据
- **查看趋势**：查看该主机指标的历史趋势图

### 3.6 配置调整

探针相关的环境变量（在 `.env` 中配置）：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `PROBE_TIMEOUT_MS` | 单次探针采集超时 | 12000（12 秒） |
| `PROBE_INTERVAL_MS` | 轮询间隔 | 15000（15 秒） |
| `PROBE_REMOTE_CONCURRENCY` | 同时采集的主机数 | 3 |

---

## 4. 删除主机

在主控台的主机卡片上点击「删除」按钮，确认后删除该主机。

删除后：
- 主机配置和关联的 SSH 凭据被清除
- 探针停止采集
- 该主机的终端会话断开
- 网站任意门链接一并删除

本机（第一张卡片）不可删除。


# 板块三：SSH 终端

---

## 1. 界面布局

1Shell 的主界面分为三个区域：

![完整工作区](sections/images/03-workspace-overview.png)

- **左侧边栏**：导航菜单（主控、程序、容器、网站、脚本、仓库、创作、AI、探针、审计）+ 主机列表 + 文件浏览器
- **中间区域**：终端主体
- **右侧面板**：AI 运维助手（可展开/收起）

---

## 2. 本地终端

登录后默认进入本机终端，即 1Shell 服务所在机器的 Shell。

![终端界面](sections/images/03-terminal-fullscreen.png)

- Windows 系统显示 CMD / PowerShell
- Linux 系统显示 Bash / Zsh
- 状态栏显示「已连接」（绿色圆点）

---

## 3. 远程 SSH 终端

在左侧主机列表中点击任意远程主机的「切换」按钮，即可 SSH 连接到该主机。

![远程终端](sections/images/03-terminal-remote.png)

连接后：
- 终端标签页自动切换到该主机
- 顶部显示主机名称和实时指标（CPU、内存、负载、磁盘）
- 终端内容为远程主机的 Shell 会话

---

## 4. 多标签页

### 4.1 标签页管理

- 每台主机对应一个终端标签页
- 点击 **「+ 新终端」** 可打开新的本地终端标签页
- 点击标签页名称切换不同终端
- 多个终端之间互不干扰，可同时在不同服务器上操作

### 4.2 多 VPS 同时管理

通过标签页可以快速在多台 VPS 之间切换。左侧主机列表显示所有主机的连接状态，点击即切换。

---

## 5. 顶部信息栏

![顶部状态栏](sections/images/03-terminal-topbar.png)

终端上方的信息栏从左到右依次显示：

| 区域 | 内容 |
|------|------|
| **主机标识** | 当前主机名称（如「本机」） |
| **实时指标** | CPU 使用率、内存使用率、负载、磁盘使用率 |
| **功能按钮** | AI Agent、1Shell AI、AI 配置、AI 展开、设置、退出 |

---

## 6. 终端工具栏

终端状态栏右侧提供以下工具按钮：

| 按钮 | 功能 |
|------|------|
| **脚本** | 打开脚本库，选择脚本注入终端执行 |
| **Playbook** | 运行一次性 AI Playbook |
| **重连** | 断线后重新连接当前会话 |
| **AI 命令** | 打开 AI 命令建议面板 |
| **清屏** | 清除终端屏幕内容 |
| **全屏** | 终端全屏显示，隐藏侧边栏和顶栏 |
| **退出** | 断开当前终端会话 |

---

## 7. 跳板机级联（ProxyJump）

![跳板机设置](sections/images/03-proxyjump.png)

当目标机器在 NAT 内网，无法从公网直接 SSH 连接时，可通过跳板机级联：

1. 先在 1Shell 中添加一台公网可达的主机作为「跳板机」
2. 添加内网目标主机时，在「中继跳板」下拉框中选择跳板机
3. 1Shell 会自动通过跳板机建立 SSH 隧道，连接到内网目标

连接路径：`1Shell → 跳板机（公网） → 目标机器（内网）`

## 8. 左侧导航栏

左侧竖向导航栏提供以下页面入口：

| 图标 | 页面 | 说明 |

| AI | AI 设置 | AI 功能配置 |
| 探针 | 探针中心 | 主机监控数据 |
| 审计 | 审计日志 | 操作记录查询 |
| 退出 | 退出登录 | 退出当前账号 |

---

## 9. 常用操作

### 9.1 复制粘贴

- **复制**：选中终端中的文本，自动复制到剪贴板（或 Ctrl+Shift+C）
- **粘贴**：Ctrl+Shift+V 或右键粘贴

### 9.2 断线重连

网络中断后终端会显示断开状态，点击工具栏的「重连」按钮恢复会话。

### 9.3 终端大小自适应

终端会自动适应浏览器窗口大小，调整窗口时终端列数和行数自动更新。


# 板块四：文件管理

---

## 1. 打开文件浏览器

文件浏览器位于左侧边栏的下半部分。点击「文件浏览」标题即可展开。

支持两种模式：
- **本地模式**：浏览 1Shell 服务所在机器的文件系统
- **远程模式（SFTP）**：浏览当前已连接的远程主机的文件系统

切换到不同主机的终端时，文件浏览器自动切换到对应主机的文件系统。

---

## 2. 目录浏览

![文件浏览器](sections/images/04-file-browser.png)

### 2.1 界面元素

- **路径栏**：显示当前目录路径（如 `/ > root`），点击路径中的任意层级可快速跳转
- **上传按钮**：上传文件到当前目录
- **返回上级**（`↑ ..`）：返回上一层目录
- **文件/文件夹列表**：黄色文件夹图标表示目录，文件图标表示文件

### 2.2 操作

### 3.1 AI 创作（推荐）

1. 进入「创作工作台」（左侧导航 → 创作）
2. 用自然语言描述需求，例如：
   - 「每 5 分钟检查所有服务器的 CPU、内存、磁盘」
   - 「监控 nginx 进程，挂了自动重启」

## 4. 实例管理

每个 Program 绑定一台或多台主机，每台主机生成一个「实例」。

### 4.1 启用实例

### 4.2 手动触发

### 4.3 自定义操作按钮

---

## 5. 三层 AI 引擎

这是 1Shell 最重要的架构设计：

### 5.1 L1：确定性执行

- 运行预设的 Shell 命令（如 `cat /proc/stat`）
- 用 `verify` 规则判断成功/失败（退出码 + 正则匹配）
- **零 AI 消耗**，纯规则驱动

### 5.2 L2：AI 约束执行

- 当 L1 发现异常时，调用 Skill（AI 能力包）
- Skill 在限定范围内诊断和修复
- 通过 `when` 条件触发，避免无谓的 token 消耗
- **少量 AI 消耗**

### 5.3 L3：Guardian AI 全权介入

- L2 也无法修复时，Guardian AI 获得完整诊断权限
- 自主分析问题根因，执行修复
- 有 `max_actions_per_hour` 滑动窗口限制，防止 AI 失控循环
- **按需 AI 消耗**

### 5.4 设计理念

能用简单规则解决的不动用 AI，需要判断力的才让 AI 介入。既省成本，又保安全。

| 格式 | 用途 |

| **keyvalue** | 键值对展示，适合单台主机的指标详情 |
| **table** | 表格展示，适合多台主机的指标对比 |
| **message** | 文本消息 |
| **list** | 列表展示 |

# 板块八：MCP 接入与仓库

---

## 1. 什么是 MCP

MCP（Model Context Protocol）是一个标准化协议，1Shell 通过 MCP 实现两个方向的 AI 协作：

- **对外暴露**：1Shell 自身作为 MCP Server，让 Claude Code 等外部 AI 工具直接调用多服务器管理能力
- **对内接入**：1Shell 可以接入外部 MCP 工具（天气、数据库、办公软件等），AI 运维过程中可调用

---

## 2. 仓库（Warehouse）

仓库是 Skill 和 MCP Server 的统一管理中心。

进入方式：左侧导航栏 → **「仓库」**

![仓库页面](sections/images/08-warehouse.png)

### 2.1 三个标签页

| 标签页 | 内容 |
|--------|------|
| **Skill** | 已有的 AI 能力包（如磁盘健康巡检、Docker 运维救援） |
| **MCP Server** | 远程 MCP Server |
| **本地 MCP** | 本地部署的 MCP Server（如 Office Word、PowerPoint） |

### 2.2 操作

# 板块九：安全体系与审计日志

---

## 1. 安全体系概览

1Shell 在赋予 AI 强大权限的同时，设计了多层安全防护机制：

| 安全机制 | 说明 |
|---------|------|
| **安全模式** | AI 的每步写操作需人工审批 |
| **登录认证** | 用户名密码 + Session 管理 |
| **凭据加密** | AES-256-GCM + scrypt 随机盐 |
| **CSRF 防护** | HttpOnly Session Cookie + JS 可读 CSRF Cookie 双 Token |
| **暴力破解防护** | 同一 IP 连续失败 5 次锁定 60 秒 |
| **时序安全** | 密码比对使用 `crypto.timingSafeEqual` |
| **AI 频率限制** | Guardian 滑动窗口 `max_actions_per_hour` |
| **安全红线** | 硬编码禁止 `rm -rf /`、`shutdown` 等破坏性命令 |
| **IP 访问控制** | 白名单 / 黑名单 |
| **API 安全** | Helmet 安全头、CSP、限流 |
| **Bridge 鉴权** | 独立 Token，与 Web Session 隔离 |

---

## 2. IP 访问控制

进入方式：设置 → **「IP 访问控制」** 标签页

![IP 访问控制](sections/images/09-ip-filter.png)

### 2.1 白名单模式

开启后，**仅允许**名单中的 IP 访问 1Shell。所有不在白名单中的 IP 请求都会被拒绝。

### 2.2 黑名单模式

开启后，**拒绝**名单中的 IP 访问。其他 IP 正常访问。

两种模式可同时开启。

### 2.3 添加规则

1. 选择类型：白名单 / 黑名单
2. 输入 IP 地址或 CIDR 格式网段（如 `192.168.1.0/24`）
3. 可选填写备注说明
4. 点击「添加」

规则列表按类型、IP/CIDR、备注三列展示，可随时删除。

---

## 3. 安全模式

安全模式是 AI 操作的人工审批机制。

### 3.1 开启方式

- 1Shell AI 面板顶部 → 点击 **「安全」** 复选框
- 或在设置中开启

### 3.2 工作流程

开启后，AI 的每一步**写操作**（执行命令、修改文件、创建资源）都会：

1. 在审批条中显示即将执行的操作内容
2. 用户点击 **「批准」** → 执行该操作
3. 用户点击 **「拒绝」** → 跳过该操作，AI 继续后续步骤

### 3.3 适用场景

- 生产环境操作，需要确认每步命令
- 初次使用 AI 功能，建立信任前的观察期
- 执行高风险操作（如修改配置、重启服务）

---

## 4. AI 安全红线

AI 硬编码禁止以下操作，即使用户或 AI 自主决策也无法绕过：

- `rm -rf /` — 删除根目录
- `shutdown` / `reboot` — 关机重启（需人工确认）
- 修改 SSH 配置 — 防止锁死远程访问
- 其他高危破坏性命令

Guardian AI 有 `max_actions_per_hour` 滑动窗口，防止 AI 在修复循环中失控。

---

## 5. 凭据加密

所有 SSH 密码和私钥使用 **AES-256-GCM + scrypt 随机盐** 加密存储：

- 每个凭据使用独立的随机盐
- 加密密钥派生自 `.env` 中的 `APP_SECRET`
- 即使数据库文件泄露，也无法还原明文凭据
- `APP_SECRET` 建议设置为 64 位以上的随机字符串

---

## 6. 反向代理配置

通过 Nginx 等反向代理访问时，需在 `.env` 中配置受信任代理 IP：

```env
TRUSTED_PROXY_IPS=127.0.0.1
```

不配置此项时，暴力破解防护获取到的客户端 IP 可能是代理 IP（如 127.0.0.1），导致所有用户共用一个 IP 限制。
